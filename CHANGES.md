# Изменения: улучшение лайков (`!likes`, `!likes shuffle`, пагинация)

Дата: 2025-12-26
Версия: 1.1.0

## Что было не так

1) При просмотре/лайках появлялась ошибка Discord embed:
`Field value length must be less than or equal to 1024 (Value)`

2) Команда `!likes` показывала только первые 10 лайков без возможности листать страницы (при тысячах лайков это бесполезно).

3) `!likes shuffle` включал один трек так, что `!n` / `!next` (пропуск) не работали: бот отвечал «Очередь пуста».

## Что сделано

### 1) Устранён лимит 1024 (embed field)

- **Причина**: `!likes` формировал список треков через `embed.AddField(...)`, а у поля embed лимит **1024 символа**.
- **Исправление**: вывод списка лайков переведён в **`embed.Description`** (лимит **4096**), плюс добавлена безопасная **обрезка длинных строк** (title/author).

### 2) Добавлена пагинация лайков

Теперь `!likes` поддерживает страницы:

- **Кнопки**: `◀/▶` листают страницы лайков прямо в сообщении.
- **Команда**: `!likes page <страница>` (или `!likes p <страница>`).

Также сохранены кнопки:

- **Shuffle**: включает режим случайного воспроизведения лайков.
- **Stop**: выключает режим лайков.
- **1..10**: проигрывание конкретного трека с текущей страницы.

### 3) `!likes shuffle` + `!n`/`!next` теперь работают корректно

- В режиме лайков **пропуск** (`!n` / `!skip` / next-кнопка в плеере) теперь **не требует очереди**.
- Если очередь пуста, бот включает **следующий случайный лайк**.
